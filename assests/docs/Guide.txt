# === DC-CRO: Diversity-Controlled CRO Implementation ===
import numpy as np
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

def compute_diversity(population):
    # Average pairwise Hamming distance between all molecule pairs
    n = len(population)
    total = 0
    for i in range(n):
        for j in range(i+1, n):
            total += np.sum(population[i] != population[j])
    return total / (n*(n-1)/2) / len(population[0])  # normalized [0,1]

def adaptive_actions(pop, KE_list, diversity, threshold=0.15):
    if diversity < threshold:
        # Action 1: Redistribute Kinetic Energy
        mean_KE = np.mean(KE_list)
        KE_list = [mean_KE * np.random.uniform(0.8, 1.2) for _ in KE_list]
        # Action 2: Inject new random molecules (crowding replacement)
        n_inject = max(1, len(pop)//5)  # replace 20% of population
        for i in range(n_inject):
            worst_idx = np.argmin(KE_list)
            pop[worst_idx] = np.random.choice([0,1], len(pop[0]), p=[0.9,0.1])
            KE_list[worst_idx] = mean_KE
    return pop, KE_list

# Main DC-CRO loop
def run_dc_cro(X_train, y_train, n_mol=20, n_iter=100, div_threshold=0.15):
    N = X_train.shape[1]
    pop   = [np.random.choice([0,1],N,p=[0.9,0.1]) for _ in range(n_mol)]
    KEs   = [0.5] * n_mol
    PEs   = [fitness(m, X_train, y_train) for m in pop]
    best  = pop[np.argmax(PEs)].copy()
    bscore= max(PEs)
    hist  = [bscore]
    div_hist = []

    for it in range(n_iter):
        # ── Standard CRO operators ──
        i = np.random.randint(n_mol)
        op = np.random.choice(['wall','decomp','inter','synth'],p=[0.5,0.2,0.2,0.1])
        candidate = apply_operator(op, pop, i, n_mol)
        score = fitness(candidate, X_train, y_train)
        if score > PEs[i]:
            pop[i], PEs[i] = candidate, score
        # ── DC-CRO: Diversity check every 10 iterations ──
        if it % 10 == 0:
            div = compute_diversity(pop)
            div_hist.append(div)
            pop, KEs = adaptive_actions(pop, KEs, div, div_threshold)
        if max(PEs) > bscore:
            bscore = max(PEs)
            best = pop[np.argmax(PEs)].copy()
        hist.append(bscore)
        if it % 10 == 0:
            print(f'Iter {it}: F1={bscore:.4f} Features={best.sum()} Diversity={div:.3f}')
    return best, bscore, hist, div_hist

SECTION 4: CRO DEEP UNDERSTANDING — EVERYTHING YOU MUST KNOW
Read this section until you can explain every concept without looking at it. Your supervisor will ask follow-up questions. These are the complete answers.

4.1  The Physical Chemistry Foundation
CRO is built on a single physical law: in a closed container, molecules collide and react until the system reaches its lowest possible energy state (thermodynamic equilibrium). Your supervisor’s papers use this thermodynamic metaphor to search for optimal solutions.

Chemistry Concept	What It Means in Chemistry	What It Means in Your Cancer Code
Closed Container	A sealed flask where reactions happen. Nothing enters or leaves.	Your training dataset. Fixed 1,837-feature space. CRO explores inside it.
Molecule	A chemical entity with a specific atomic arrangement.	A binary array of 1,837 bits. The specific arrangement of selected genes.
Potential Energy (PE)	Energy stored in chemical bonds. High PE = unstable molecule. Reactions seek minimum PE.	PE = 1 − F1_weighted. Maximum PE = worst classifier. Algorithm seeks minimum PE = best classifier.
Kinetic Energy (KE)	Energy of motion. Molecules with more KE can overcome energy barriers (activation energy).	Tolerance buffer. High KE lets the algorithm temporarily accept a worse feature set to escape local optima.
Thermodynamic Equilibrium	Final state where no more net energy change occurs. The system has found its minimum.	Convergence. When fitness stops improving across 10+ consecutive iterations, CRO has found the optimal feature subset.

4.2  The Four Reactions — Deep Explanation
Reaction 1: On-Wall Ineffective Collision (Local Search)
When a molecule hits the container wall, it bounces back with slightly altered structure but loses some energy (ineffective = energy lost to heat). The molecule mostly stays intact but one atom changes position.
  IN YOUR CODE
A feature vector flips 1–5 randomly chosen bits. Gene that was selected (1) becomes unselected (0) and vice versa.
This tests: ‘What if we drop gene X and add gene Y instead? Does accuracy improve?’
Use case: Fine-tuning a good solution. Happens 50% of iterations. This is exploitation.
Energy rule: If new PE (1 − F1) < old PE, accept. If worse, only accept if KE buffer allows it.

Reaction 2: Decomposition (Escaping Local Optima)
A molecule absorbs so much energy it cannot hold its structure. It violently splits into two separate, smaller molecules. Both children must obey energy conservation: PE(child1) + PE(child2) ≤ PE(parent) + KE(parent).
  IN YOUR CODE
A noisy 1,837-bit array is split into two sparser arrays. Forces the model to try simpler gene combinations.
This is why it ‘cures the curse of dimensionality’ — it breaks massive, bloated feature sets into leaner ones.
Use case: When a molecule is stuck with too many mediocre genes. Triggered when diversity drops. This is exploration.
Critical insight: Two children together may rediscover better combinations than the cluttered parent.

Reaction 3: Inter-Molecular Ineffective Collision (Crossover)
Two molecules collide. They exchange atoms but neither fully merges. Both exit the reaction changed but still distinct. Energy is conserved across both products.
  IN YOUR CODE
Two binary arrays swap 50% of their bits using a random crossover mask.
Gene combination from molecule A meets gene combination from molecule B.
Use case: When two different good solutions exist. Sharing genes may create a better hybrid.
Biological meaning: Perhaps the RNA-seq markers from one solution combine well with the protein markers of another.

Reaction 4: Synthesis (Global Convergence)
Two highly stable molecules (low PE) fuse completely into one. Synthesis only happens when both reactants have very low energy — meaning both are already good solutions. The product is a super-molecule combining the best features of both.
  IN YOUR CODE
np.bitwise_or(sol1, sol2) — the union of two good feature sets.
All genes selected by EITHER molecule are retained in the super-molecule.
Use case: Late iterations when two near-optimal solutions exist. Creates the final best answer.
Warning: Synthesis can create a large molecule (too many genes). The parsimony term in the fitness function
penalizes this — so the model is pushed to drop non-contributing genes in subsequent collisions.


4.3  The DC-CRO Addition — Diversity Monitoring
This is the specific enhancement from your supervisor’s 2025 thesis. Standard CRO has no mechanism to detect when it is stuck. DC-CRO adds a watchdog.

DC-CRO Component	What It Does in Your Pipeline
Diversity Metric	Compute average pairwise Hamming distance between all molecule pairs. If molecules are nearly identical (diversity < 0.15), diversity is low.
KE Redistribution	When diversity drops below threshold, normalize and randomize KE values. Some molecules get more energy to escape current local region.

  Q1: WHY CRO AND NOT A GENETIC ALGORITHM OR PSO FOR FEATURE SELECTION?
A: ‘Genetic Algorithms use only crossover and mutation — no mechanism to detect or prevent premature
   convergence. PSO uses velocity-based clustering that fails in 1,837 dimensions. As my supervisor’s
   2025 thesis shows, standard CRO also suffers from this. That is why I used DC-CRO: it adds a
   diversity monitor and adaptive KE redistribution that neither GA nor PSO have. The experimental
   comparison shows DC-CRO achieves [X]% higher F1 than standard CRO on the same data.’

  Q2: HOW DO YOU PREVENT DATA LEAKAGE BETWEEN TRAINING AND TEST?
A: ‘Strict procedural isolation. The test set is created first via stratified 80/20 split before any
   preprocessing. SMOTE is fitted exclusively on training data and applied only to training data.
   Feature scalers are fitted on training data only, then the transform-only is applied to test data.
   DC-CRO feature selection uses only cross-validation within the training set. The test set is
   evaluated exactly once — at the very end, after all selection and tuning is complete.’

  Q3: WHAT GUARANTEES CRO FINDS THE GLOBAL OPTIMUM AND NOT JUST A LOCAL ONE?
A: ‘CRO provides no deterministic guarantee of the global optimum — no metaheuristic does, by
   definition (NP-Hard). What CRO does guarantee is better approximation than exact search by
   efficiently sampling the 2^1837 space. The DC-CRO diversity mechanism specifically reduces the
   probability of premature convergence to a local optimum by injecting new solutions when clustering
   is detected. The convergence curve shows fitness still improving at iteration 80+ — evidence the
   algorithm is still exploring, not stuck in a local optimum.’

  Q4: WHY SMOTE — WHAT ARE ITS LIMITATIONS?
A: ‘SMOTE generates synthetic minority samples by interpolating between real minority patients in
   feature space. It is necessary here because HER2+ patients are at 5.3:1 imbalance — without
   correction, any classifier predicts ‘negative’ for everyone and appears 84% accurate while
   clinically useless. Limitations: SMOTE assumes smooth feature space (real genomics has discontinuities),
   may generate biologically implausible synthetic patients, and slightly inflates minority-class
   precision. This is why I report MCC alongside F1 — MCC is robust to SMOTE-induced artifacts.’

  Q5: WHAT DOES YOUR SHAP PLOT ACTUALLY PROVE?
A: ‘SHAP assigns each feature a contribution value (Shapley value from cooperative game theory)
   for each individual prediction. It does not just say ‘ESR1 is globally important’ — it says
   ‘ESR1 pushed Patient 42’s prediction toward ER+ by 0.42 and Patient 17’s toward ER− by 0.11.’
   What the plots prove: DC-CRO selected biologically correct features — ESR1 for ER, ERBB2 for
   HER2, CDH1 for lobular histotype. These are validated by 60+ years of clinical oncology literature.
   This is not an accident. It is evidence that the algorithm discovered real biological signal.’

  Q6: WITH ONLY 705 SAMPLES, CAN YOU TRUST THESE RESULTS?
A: ‘705 is the complete TCGA BRCA multiomics cohort. There is no larger fully-profiled dataset
   across all four omics layers simultaneously. The 5-fold stratified cross-validation and mean ±
   std reporting quantify variance across data arrangements. The SHAP biological alignment provides
   external validation — the selected features match published clinical biomarkers, not random noise.
   SMOTE compensates for minority-class sparsity. Limitations of sample size are stated explicitly
   in the Discussion. External validation on an independent cohort is listed as future work.’

  Q7: WHAT HAPPENS IF CRO SELECTS DIFFERENT FEATURES ON A DIFFERENT RANDOM SEED?
A: ‘This is a real limitation of all stochastic search methods, which I acknowledge in Discussion.
   The mitigation is twofold: (1) I fix the random seed (42) for all stochastic components for
   full reproducibility. (2) The SHAP biological validation acts as a stability check — if the
   same key biomarkers (ESR1, ERBB2) appear across runs with different seeds, it confirms the
   selection is robust. Multi-seed analysis with mean ± std across seeds is listed as future work.’

  Q8: HOW IS THIS CLINICALLY APPLICABLE IN PRACTICE?

SECTION 8: QUICK REFERENCE — TERMINOLOGY, EQUATIONS & CHECKLISTS

8.1  Key Terminology from Supervisor’s Papers
Term	Definition (from his papers)
NP-Hard	Non-deterministic Polynomial-time Hard. A class of problems where no algorithm can find the exact optimum in polynomial time. 2019 survey: ‘Exact algorithms fail to solve most of them in polynomial time.’
Metaheuristic	A high-level strategy for searching large solution spaces by approximating global optima. CRO is a metaheuristic.
Local Optimum	A solution better than all nearby solutions but not the best possible globally. Like finding the tallest hill in one valley but missing a higher mountain elsewhere.
Global Optimum	The single best solution across the entire search space. In your context: the feature subset with the highest possible F1 score.
Premature Convergence	From his 2025 thesis: ‘The algorithm tended to converge too fast, which meant that the population lost its diversity.’ Molecules cluster around a local optimum and stop exploring.
Population Diversity	How different the molecules are from each other. High diversity = molecules exploring many different feature subsets. Low diversity = all molecules similar (dangerous).
Kinetic Energy (KE)	The exploration buffer in CRO. Allows molecules to temporarily accept worse solutions. When KE depletes, the molecule can only accept improvements (exploitation mode).
Exploration vs Exploitation	Exploration: searching new, untested areas (Decomposition, random injection). Exploitation: refining known good solutions (On-wall, Inter-molecular collision).
DC-CRO	Diversity-Controlled CRO. From his 2025 thesis. Adds a diversity monitor and 4 adaptive actions to prevent premature convergence in high-dimensional spaces.

8.2  All Key Equations
Fitness:       f(S) = 0.9 × F1_weighted(S) − 0.1 × (|S| / N)
CRO Energy:    E(S) = 1 − f(S)    [minimized by algorithm]
Diversity:     D = avg pairwise Hamming(sol_i, sol_j) / N   [0=identical, 1=opposite]
SMOTE:         x_new = x_i + λ × (x_nn − x_i)   where λ ~ Uniform(0,1)
MCC:           (TP×TN − FP×FN) / sqrt((TP+FP)(TP+FN)(TN+FP)(TN+FN))
F1-weighted:   weighted average of F1 per class by support
Search space:  2^1837 ≈ 10^553   [incomprehensibly larger than atoms in universe ~10^80]
